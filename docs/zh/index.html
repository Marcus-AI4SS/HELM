HELM

HELM 是本地研究项目的指挥看板。它帮你查看项目状态、证据准备、已有文件、本地检查，以及下一步要交给 Codex 的说明。

统一产品语言

VELA 和 HELM 是一套工作流里的两个分工，不是同一个工具。

VELA Versatile Experiment Lab & Automation

准备可携带的项目结构和有边界的 Codex 工作。

HELM Hub for Evidence, Logs & Monitoring

读取本地状态，并把它变成清楚的项目看板。

HELM 帮你看什么

HELM 只负责把项目看清楚，不把 app 变成隐藏研究执行器。

项目状态

查看当前项目、阶段、阻塞、最近活动，以及项目文件夹能不能读取。

证据准备

看哪些已经读到、哪些缺失、哪些还需要回到 Codex 继续检查。

文件和产物

从一个地方打开已有报告、笔记、图表、表格和本地检查记录。

给 Codex 的说明

复制有边界的下一步说明，不让 HELM 自动执行研究任务。

HELM 和 VELA 怎么协作

VELA 不需要 HELM 也能安装使用。HELM 不需要 VELA 也能运行。组合使用时，两者通过显式本地文件交换状态，而不是依赖隐藏 app 状态。

VELA → HELM

VELA 写出项目上下文。HELM 读取 vela.project.context.v1，展示路径、状态、证据计数、文件、阻塞和本地检查。

HELM → Codex / VELA

HELM 准备可复制的 helm.codex.handoff.v1 说明。只有用户显式保存或导出时，VELA 才应写入项目。

1. 安装或构建

3. 复制给 Codex

在“给 Codex 的说明”里复制下一步，把研究推进放回 Codex。
