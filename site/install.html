Install

Build HELM from source today. Signed releases come next.

HELM is in public-candidate status. The source build is ready; signed Windows and macOS artifacts are tracked in the release checklist before the first tagged public release.

Windows

Use the public candidate installer when it is attached to a release. Until then, build locally with the commands below.

macOS

Use the same HELM source repository. The app reads your own local folders; it does not need the maintainer's private environment.

After opening

If HELM has no trusted project, copy the Project page intake template and paste it into Codex with your project path.

1 Install dependencies 
cd apps/desktop
npm install
2 Build and check 
npm run build
cd src-tauri
cargo check
3 Run the public gate 
powershell -ExecutionPolicy Bypass -File release/check-public-release.ps1

Build local artifacts

The artifact script records signing readiness and SHA-256 checksums. It does not pretend unsigned artifacts are signed.

powershell -ExecutionPolicy Bypass -File release/build-public-artifacts.ps1
