HELM

HELM is the local command board for research projects. It shows status, evidence readiness, existing files, local checks, and the note you should take back to Codex.

Shared product language

VELA and HELM are one workflow pair with two separate jobs.

VELA Versatile Experiment Lab & Automation

Prepares portable project structure and bounded Codex work.

HELM Hub for Evidence, Logs & Monitoring

Reads local state and turns it into a clear dashboard.

What HELM Helps You See

HELM keeps the project visible without turning the app into a hidden research engine.

Project Status

See the active project, stage, blockers, recent activity, and whether the folder can be read.

Evidence Readiness

Check what has been read, what is missing, and what still needs Codex review.

Files And Outputs

Open existing reports, notes, figures, tables, and local check records from one place.

Codex Notes

Copy a bounded next-step note for Codex without asking HELM to execute the work.

How HELM And VELA Work Together

VELA can be used without HELM. HELM can run without VELA. Together they exchange explicit local files rather than hidden app state.

VELA → HELM

VELA writes project context. HELM reads vela.project.context.v1 to show paths, status, evidence counts, files, blockers, and local checks.

HELM → Codex / VELA

HELM prepares helm.codex.handoff.v1 as a copyable Codex note. VELA stores it only after an explicit user save or export.
