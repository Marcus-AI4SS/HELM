HELM 本地项目看板

看清项目状态，再回到 Codex 推进研究。

HELM 读取电脑上的项目状态、证据准备、已有文件和环境健康。 它不接管研究过程，只帮你判断哪里已准备好、哪里被阻塞、接下来该把什么交给 Codex。

VELA 可以准备项目事实、证据痕迹、校验结果和运行状态。

HELM

读取并展示项目状态

HELM 把这些本地状态整理成页面，并给出可复制的 Codex 继续说明。

首次使用

没有项目时，从复制接入说明开始。

HELM 里不用聊天，也不用新建项目。Codex 负责连接项目文件夹并准备上下文；HELM 负责刷新后显示结果。

  1. 打开 HELM先看“项目”页面是否已经出现项目。
  2. 复制接入说明没有项目时，复制“项目”页面给出的说明。
  3. 交给 CodexCodex 连接项目后，回到 HELM 刷新并查看五个页面。

边界

HELM 是看板，不是研究引擎。

  • 本仓库没有托管遥测或云同步。
  • HELM 不提供提示词输入、聊天窗口或隐藏任务调度。
  • 公开包不应包含个人项目、笔记库、浏览器状态或文献数据库。
  • 没有 VELA 时，HELM 可显示空状态或公开安全示例状态。
HELM 负责展示本地项目状态。 研究推进仍回到 Codex。
