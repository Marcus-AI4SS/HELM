HELM

用 HELM 查看项目状态、证据、交付物、环境健康和下一步 Codex 交接。它可以独立运行，也可以在 VELA 环境存在时读取 VELA 项目上下文。

构建 HELM 导入 VELA VELA 仓库 GitHub

HELM 提供什么

在不把 app 变成隐藏研究执行器的前提下，给本地研究状态一个可读看板。

项目状态

展示阶段、阻塞、近期活动和活跃文件，不把项目迁入 app 内部。

证据准备度

展示证据等级和来源计数，不把文件读取夸大成事实核验。

交付物

索引磁盘上已有的报告、笔记、图表和交接输出。

Codex 交接

准备包含任务、文件、约束、输出、缺口和复核标准的有边界交接包。

两个独立品牌，一套共享契约

VELA 不需要 HELM 也能安装使用。HELM 不需要 VELA 也能运行。组合使用时，两者通过显式本地文件交换状态，而不是依赖隐藏 app 状态。

VELA 到 HELM

HELM 导入 vela.project.context.v1，读取项目路径、状态、证据计数、交付物、环境健康和交接准备度。

HELM 到 VELA

VELA 导入 helm.codex.handoff.v1，作为有边界的 Codex 交接包存入项目 handoffs/ 层。

HELM 是本地科研看板。VELA 是可移植科研工作流环境。
