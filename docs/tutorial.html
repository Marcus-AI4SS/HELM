First-run guide

Open HELM and know what to click.

HELM is a local dashboard. It does not chat, write, create projects, or run research for you. It helps you see local project status and copy the right continuation note back to Codex.

If You Do Not Know Where To Go

Pick what you want to do. HELM keeps the main workflow on five pages.

I have no project yet

Open Project, click Copy setup instructions, then paste them into Codex with your real project folder.

I want to switch projects

Use Current Project at the top, or choose a readable project from the Project page.

I want to check materials

Open Evidence to see whether materials and evidence records are readable.

I need Codex to continue

Open Give to Codex, copy the prepared note, and send it in Codex.

I want to open files

Open Files to view documents, reports, figures, tables, and packages already on disk.

Something looks broken

Open Local Machine, run a local check, or copy the diagnostic note for Codex.

First Run In Six Steps

1. Open Project

Start on the Project page. If HELM already sees your project, check its current status.

2. Copy setup instructions

If there is no project, copy the setup instructions. This only copies text and does not change files.

3. Go to Codex

Paste the instructions into Codex and add the real project folder path.

4. Return and refresh

After Codex prepares the local context, return to HELM and click Refresh.

5. Read the five pages

Project, Evidence, Give to Codex, Files, and Local Machine each answer one part of the workflow.

6. Continue in Codex

When you need the next step, copy the note from Give to Codex and continue there.
